1. HTML
  - 웹문서의 구조
 
2. CSS
  - 웹문서의 스타일(색, 크기, 레이아웃 등)

3. JavaScript
  - 웹문서의 동작(제어)
  - '자바스크립트'와 '자바'는 완전히 다른 언어
  ◈ jQuery, React, Vue 등은 "자바스크립트 라이브러리"



❤ HTML
 - Hyper Text Markup Language
 - 팀 버너스 리가 최초로 제안(고안)한 문법
 - 웹문서를 구성하는 요소(문법)를 태그(tag)라고 함
 - 태그는 꺽쇠 괄호(<,>) 안에 태그 이름을 작성하여 코드화 함
   (예시 설명) 단락을 나타내는 태그는 p 태그
   (예)<p>
 - 태그는 일반적으로 시작태그와 종료태그 한 세트로 구성됨
   : 종료 태그는 </태그이름>
  (예) <p>단락은 영어로 paragraph 입니다.</p>
 - 간혹 종료태그가 없는 빈(empty) 태그도 존재
  <br>, <hr>, <meta>, <link>, <img>, ....


- 태그는 속성(attribute)을 갖기도 함
- 거의 대부분의 속성은 값(value)을 갖음
- 속성과 속성값은 띄어쓰기 없이 =(등호)로 연결
- 속성값은 ""(큰따옴표)를 사용


<인간 다리갯수="2" 날개유무="무" 종속목="포유류">
<강아지 다리갯수="4" 날개유무="무" 종속목="포유류">
<닭 다리갯수="2" 날개유무="유" 종속목="조류">

✔웰코딩(Well-Coding)
- 브라우저에는 아무런 영향을 주지 않음
- 개발자의 가독성을 위해  반드시 필요
- 오타와 버그의 확률 감소

1. 시작태그를 작성했다면 반드시 종료태그부터 작성한다.
2. 자손요소는 부모요소보다 1tab 들여쓴다
3. 시작태그와 종료태그는 같은 위치에 작성한다.(들여쓰기 X)
4. 형제관계는 같은 위치에 작성
5. 태그명, 속성명은 모두 소문자로 작성
