구조 분할, 그룹

의미 없는 분할(그룹)

div

  • division(나누다) => Block 속성
  • Block 요소들을 의미 없이 나누거나 그룹
  • 화면 구조(요소들의 배치)와 비슷한 목적을 가진 요소들을 그룹할 목적
  • 하나의 태그가 아니라 여러 태그를 그룹할 때 사용
  • 꼭 필요한 경우에만 사용
  • 의미 요소(제목, 단락 등)에 사용 금지

span

  • 인라인 요소
  • 줄이 바뀌면 안되는 인라인 요소(주로 텍스트)를 나누거나 분할

의미를 부여한 분할

  • Sementic 태그
  • 레이아웃 구성을 위해 단순히 구역을 나누던 div에 영역의 의미를 부여한 태그
  • html5에 추가됨
header
- 웹문서 상단에 로고, GNB, 검색 등을 포함하는 영역
- 일반적으로 문서 전체의 머리글로 사용되지만, 섹션의 머리글을 표시할 때도 사용
footer
- 웹문서 하단에 저작권 표시, 저작자/소유자의 연락 정보, 패밀리 사이트 등을 포함하는 영역
- 문서 전체 뿐만 아니라 섹션의 바닥글을 표시할 때도 사용
main
- 웹문서의 본문에 해당하는 영역에 사용
- 하나의 문서에서 단 한번만 사용
section
- 문서 내에서 다른 주제를 가진 영역을 구분
- 주제를 대변하는 heading 태그(h2~h6)가 필요
- section 태그는 header의 부모태그가 될 수 있다.
article
- (신문·잡지의) 글, 기사 / 블로그의 포스팅글 등
- 독립적인 컨텐츠, 다른 사이트 등에 배포가 가능
- section의 자손이 될 수 있고, 부모 요소도 될 수 있음
aside
- 본문의 내용(문서의 가장 주요한 내용)과 직접적으로 관련이 없는 내용
nav
- 내비게이션 영역을 표시
- 내비게이션 : 웹사이트의 페이지들의 분류와 구조를 파악할 수 있는 링크 목록
- 단순한 링크와는 다름
hgroup
- 제목인 hn 태그와 제목을 부가적으로 설명하는 글 을 하나로 그룹해주는 역할
figure
- 이미지 등 생략해도 주요 내용에 영향을 미치지 않는 콘텐츠
- figcaption 태그를 자손으로 가지고 figure의 내용을 설명
